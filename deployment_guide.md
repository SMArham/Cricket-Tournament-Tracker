# Step-by-Step Deployment Guide

This document outlines the step-by-step instructions to deploy the **Cricket Tournament Tracker** application online using **Supabase** (Database), **Render** (Flask Backend), and **Vercel** (Static HTML/JS Frontend).

---

## Phase 1: Push Project to GitHub

Before deploying online, you need to push your local code to a GitHub repository.

1. Go to [GitHub](https://github.com) and log in.
2. Click **New** to create a new repository.
   - **Repository name**: `cricket-tournament-tracker` (or any name you prefer)
   - **Public/Private**: Select according to your preference.
   - **Initialize repository**: Do **NOT** add a README, `.gitignore`, or license (we already have them).
   - Click **Create repository**.
3. Open a terminal/command prompt in the **project root directory** (`ML_Cricket_Tracker`) on your computer and run the following commands:

   ```bash
   # Initialize Git repository
   git init

   # Add all files (the .gitignore will automatically prevent sending large model binaries or secret .env files)
   git add .

   # Commit files locally
   git commit -m "Initial commit: Ready for deployment"

   # Rename branch to main
   git branch -M main

   # Add your GitHub repository link as remote origin (replace with your actual link)
   git remote add origin https://github.com/YOUR_GITHUB_USERNAME/cricket-tournament-tracker.git

   # Push code to GitHub
   git push -u origin main
   ```

---

## Phase 2: Setup Database in Supabase

Your application uses Supabase to authenticate users and store prediction history.

1. Log in to [Supabase](https://supabase.com).
2. Select your project.
3. In the left-hand sidebar, click on **SQL Editor**.
4. Click **New Query** (blank query).
5. Paste the following SQL script in the editor to create the necessary tables with correct relationships and schemas:

   ```sql
   -- 1. Create the Users Table
   CREATE TABLE IF NOT EXISTS users (
       email TEXT PRIMARY KEY,
       password TEXT NOT NULL,
       created_at TIMESTAMPTZ DEFAULT NOW()
   );

   -- 2. Create the Predictions Table
   CREATE TABLE IF NOT EXISTS predictions (
       id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
       user_email TEXT REFERENCES users(email) ON DELETE SET NULL,
       chasing_team TEXT NOT NULL,
       defending_team TEXT NOT NULL,
       target_score INTEGER NOT NULL,
       current_score INTEGER NOT NULL,
       wickets_lost INTEGER NOT NULL,
       balls_left INTEGER NOT NULL,
       chasing_probability DOUBLE PRECISION NOT NULL,
       defending_probability DOUBLE PRECISION NOT NULL,
       predicted_winner TEXT NOT NULL,
       created_at TIMESTAMPTZ DEFAULT NOW()
   );

   -- 3. Disable Row Level Security (RLS) for testing or create custom policies
   ALTER TABLE users DISABLE ROW LEVEL SECURITY;
   ALTER TABLE predictions DISABLE ROW LEVEL SECURITY;
   ```

6. Click the **Run** button at the bottom right. You should see a message confirming the query ran successfully.

---

## Phase 3: Deploy Flask Backend on Render

Render is a cloud hosting platform suitable for running your Python Flask API.

1. Sign up/log in to [Render](https://render.com).
2. In the dashboard, click **New +** (top right) and select **Web Service**.
3. Select **Build and deploy from a Git repository** and click **Next**.
4. Connect your GitHub account and click **Connect** next to the `cricket-tournament-tracker` repository.
5. In the creation form, fill out these fields:
   - **Name**: `cricket-tracker-backend` (or similar name)
   - **Region**: Choose the region closest to your audience (e.g., Oregon or Singapore)
   - **Branch**: `main`
   - **Root Directory**: Leave it blank (defaulting to repository root `/`)
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r backend/requirements.txt`
   - **Start Command**: `gunicorn backend.app:app`
6. Scroll down and click **Advanced** -> **Add Environment Variable**. Add your Supabase credentials so the backend can communicate with the database:
   - Key: `SUPABASE_URL` | Value: (Copy the Supabase Project URL from your local `.env` file)
   - Key: `SUPABASE_KEY` | Value: (Copy the Supabase Secret Key from your local `.env` file)
7. Click **Create Web Service** at the bottom of the page.
8. Wait for the build process to finish. It will install dependencies, load the ML models, and boot the server. Once it succeeds, copy your live backend URL displayed at the top left (e.g., `https://cricket-tracker-backend.onrender.com`).

---

## Phase 4: Point Frontend to Render Backend

Now that your backend is live, you must update the frontend code to call your live Render API instead of the local server.

1. Open the file `frontend/script.js` in your editor.
2. Go to line 3. Replace the placeholder URL with your **live Render backend URL** that you copied in Phase 3:

   ```javascript
   // Change from:
   const API_BASE = (window.location.hostname === 'localhost' || window.location.hostname === '127.0.0.1')
       ? 'http://127.0.0.1:5000'
       : 'https://YOUR-BACKEND-APP-NAME.onrender.com';

   // To (Example):
   const API_BASE = (window.location.hostname === 'localhost' || window.location.hostname === '127.0.0.1')
       ? 'http://127.0.0.1:5000'
       : 'https://cricket-tracker-backend.onrender.com'; // <-- Replace with your real Render URL
   ```

3. Save the file.
4. Commit and push this updated file to your GitHub repository:

   ```bash
   git add frontend/script.js
   git commit -m "Update production API URL to Render backend"
   git push origin main
   ```

---

## Phase 5: Deploy Frontend on Vercel

Vercel is a global cloud platform specialized for hosting fast static frontend websites.

1. Sign up/log in to [Vercel](https://vercel.com).
2. Click **Add New** -> **Project**.
3. Find your `cricket-tournament-tracker` repository under the imported list and click **Import**.
4. In the project setup panel:
   - **Framework Preset**: Select **Other**.
   - **Root Directory**: Click the **Edit** button next to it, select the `frontend` folder, and click **Continue**. (This ensures Vercel only deploys the static files inside the `frontend` folder instead of the entire repo).
   - Leave the **Build and Output Settings** unchanged (blank defaults).
5. Click **Deploy**.
6. Within a few seconds, your site will be live! Vercel will provide you with a production URL (e.g., `https://cricket-tournament-tracker-five.vercel.app`).
